# Exp-A 结果备忘录：Trial-Level 动态 Gate 最小版

**日期**: 2026-03-07  
**实验目录**: `results/stage3/trial_dynamic_gate_exp_a/2026-03-07-trial-dynamic-exp-a`  
**实验定位**: 阶段三中用于验证“trial-level 动态权重本身是否站得住”的最小原型，不作为论文主结果。

---

## 1. 实验目的

Exp-A 的目标不是直接拿到论文级最优性能，而是回答三个更基础的问题：

1. **trial-level 的连续权重 `w_t` 是否会学出有意义的变化？**
2. **这些变化是否与最简单且最强的几何信号 `d_tgt` 有合理关系？**
3. **在只用 `d_tgt` 的前提下，dynamic gate 是否至少开始接近固定 `w` 基线？**

本轮配置严格保持最小化：

- 基座：`RA + CSP + LDA`
- 对齐器：协方差空间标量插值  
  `C'_t = (1 - w_t) C_t + w_t RA(C_t)`
- gate 输入：`d_tgt` only
- 无 `KCAR`
- 无 feedback
- 无伪标签更新

---

## 2. 主要结果

### 2.1 平均性能

| 方法 | mean accuracy | std |
|------|---------------|-----|
| fixed `w=1.0` | **0.4340** | 0.1406 |
| dynamic | **0.4147** | 0.1333 |
| fixed `w=0.5` | 0.4101 | 0.1352 |
| fixed `w=0.0` | 0.3804 | 0.1179 |

### 2.2 直接比较

- dynamic **明显优于** `fixed w=0.0`
- dynamic **略优于** `fixed w=0.5`
- dynamic **明显落后于** `fixed w=1.0`

换句话说，这个最小版 dynamic gate 已经不是“完全无用”，但离“超过最强 fixed baseline”还差一截。

---

## 3. 被试级现象

### 3.1 dynamic vs fixed `w=1.0`

dynamic 相对 `fixed w=1.0`：

- 更好：`A04`, `A05`
- 持平：`A02`
- 更差：`A01`, `A03`, `A06`, `A07`, `A08`, `A09`

这说明当前最小 gate 还**不具备稳定替代强对齐的能力**。

### 3.2 dynamic vs fixed `w=0.5`

dynamic 相对 `fixed w=0.5`：

- 更好：`A01`, `A03`, `A04`, `A05`, `A06`, `A07`, `A08`
- 持平：`A02`
- 更差：`A09`

这说明 dynamic gate 的确在做一点“比常数 `0.5` 更细”的决策，但这种改进幅度还不足以追上 `fixed w=1.0`。

---

## 4. `w_t` 结构分析

### 4.1 有信息的一面

- `w_t` 与 `d_tgt` 在 **9/9 个被试** 上都呈正相关
- 相关系数大致在 `0.49 ~ 0.72` 之间

这说明：

> gate 至少学到了正确的方向：当 target 偏离更大时，应该倾向更强对齐。

### 4.2 不足的一面

虽然方向是对的，但幅度非常小：

- 平均 `w_mean = 0.5333`
- 平均 `w_std = 0.0056`
- 所有被试的 `w_t` 都只在一个很窄的区间内波动

实际观察上，它更像：

> “一个围绕 `0.53` 轻微抖动的近常数权重”

而不是：

> “真正有判别性的 trial-level 动态控制器”

这就是本轮最重要的事实。

---

## 5. 这轮可以下什么结论

### 可以下的结论

1. **trial-level dynamic gate 不是完全空的。**
   - 它没有乱学
   - `w_t` 与 `d_tgt` 的关系方向是对的

2. **`d_tgt`-only gate 已经略微优于固定 `w=0.5`。**
   - 说明 per-trial gate 并非完全没价值

3. **当前 dynamic gate 还不足以挑战强默认策略 `fixed w=1.0 / RA`。**

### 不能下的结论

1. 不能说：trial-level online dynamic alignment 已经被证明成立
2. 不能说：最小版 gate 已经值得直接叠加更复杂控制器
3. 不能说：这条线已经有足够证据进入顶会/顶刊主叙事

---

## 6. 为什么会得到这样的结果

当前最合理的解释是：

### 6.1 `d_tgt` 作为单一信号只够告诉模型“方向”，不够告诉模型“力度”

它能让 gate 学到：

- 偏离更大 → `w_t` 应更大

但它不足以让 gate 学到：

- 哪些 trial 应该非常接近 `1.0`
- 哪些 trial 应该明显回落到 `0.5` 以下

于是模型学成了一个**温和且近常数的 gate**。

### 6.2 强默认策略仍然是 RA

`fixed w=1.0` 的平均表现最好，说明在当前 setting 里：

> 大多数 trial 仍然更接近“应当强对齐”而不是“应当显著偏离”

因此最小版 dynamic gate 想超越 `RA`，需要更丰富的几何上下文，而不仅是 `d_tgt` 一个标量。

### 6.3 这轮失败不是“主意错了”，而是“最小版信号太弱”

这点很关键。

因为如果 `w_t` 与 `d_tgt` 关系混乱、甚至负相关，那才更像是思路有根本问题。  
但现在不是这样。

现在的问题更像：

> gate 学会了正确方向，但还没有足够的信息去做足够大的、足够有辨识度的 trial-level 调整。

---

## 7. 是否继续到 Exp-B？

### 我的判断：**应该继续**

原因不是因为 Exp-A 已经成功，而是因为它满足了“结构优先”的继续条件：

- `w_t` 没有完全失去结构
- `w_t` 与 `d_tgt` 关系合理
- dynamic 没有全面崩溃
- dynamic 相比 `fixed w=0.5` 有小幅正收益

因此最合理的下一步是：

## **Exp-B：trial-level 动态 gate 几何版**

输入升级为：

- `d_src`
- `d_tgt`
- `sigma_recent`

保持其余部分不变，看 richer geometry 是否能把当前“近常数 gate”拉开。

---

## 8. 当前不建议做的事

在 Exp-A 之后，**还不建议**直接做：

- 把 `KCAR` 直接塞回主控器
- 继续扩窗口级 policy 作为主线
- 直接加 feedback
- 上更复杂 MLP / 仿射变换

原因很简单：

> 我们还没有把最小的 trial-level 动态 gate 先跑通。

---

## 9. 正确的下一步定位

当前阶段三最合理的逻辑链应该是：

1. **Exp-A**：确认 trial-level `w_t` 本身能否学出结构  
2. **Exp-B**：确认 richer geometry 能否让 `w_t` 不再塌成近常数  
3. **Exp-C**：把已经验证过的窗口级 `rho_window` 当作慢变量注入 gate  
4. **Exp-D**：最后再把最简单 feedback 接回来

因此这轮 Exp-A 的价值在于：

> 它没有证明主线已经成功，但证明了 trial-level 动态 gate 这条路还值得继续，而且下一步最该补的是“更丰富的几何上下文”，不是更复杂的控制器。
