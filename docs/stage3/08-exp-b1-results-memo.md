# Exp-B.1 结果备忘录

**日期**: 2026-03-07  
**实验目录**: `results/stage3/trial_dynamic_gate_exp_b1/2026-03-07-trial-dynamic-exp-b1`

---

## 1. 实验目标

Exp-B.1 的目标非常单纯：

> 只改 `d_src / d_tgt / sigma_recent` 的输入尺度，不改 gate 结构、不改参数，验证 Exp-B 的失败是否主要来自尺度不匹配。

因此它是一个**归因实验**，不是追最优结果的实验。

---

## 2. 主要结果

### 2.1 平均性能

| 方法 | mean accuracy | std |
|------|---------------|-----|
| fixed `w=1.0` | **0.4340** | 0.1406 |
| fixed `w=0.5` | 0.4101 | 0.1352 |
| **dynamic_exp_b1** | **0.4074** | 0.1452 |
| fixed `w=0.0` | 0.3804 | 0.1179 |

### 2.2 与 Exp-A / Exp-B 的直接比较

Exp-B.1 相对 Exp-A：

- `accuracy_mean_delta = -0.00733`
- `wins_vs_exp_a = 2`
- `losses_vs_exp_a = 7`
- `w_std_mean_delta = +0.08253`

Exp-B.1 相对 Exp-B：

- `accuracy_mean_delta = -0.00617`
- `wins_vs_exp_b = 4`
- `losses_vs_exp_b = 5`
- `w_std_mean_delta = +0.08242`

这说明：

1. 性能层面，Exp-B.1 **比 Exp-A / Exp-B 更差**
2. 结构层面，Exp-B.1 **比 Exp-A / Exp-B 强得多**

---

## 3. 与 fixed 基线的关系

dynamic_exp_b1 相对 fixed 基线：

- vs `fixed w=1.0`
  - `wins = 1`
  - `losses = 8`
  - `mean_delta = -0.0266`
  - `worst_delta = -0.0938`

- vs `fixed w=0.5`
  - `wins = 1`
  - `losses = 6`
  - `draws = 2`
  - `mean_delta = -0.0027`
  - `worst_delta = -0.0208`

- vs `fixed w=0.0`
  - `wins = 8`
  - `losses = 1`
  - `mean_delta = +0.0270`

结论很明确：

> Exp-B.1 已经不再只是“比 fixed `0.5` 略好一点的动态版本”，而是掉到了 **接近 fixed `0.5` 但整体略差** 的位置。

---

## 4. 结构分析：归一化确实修复了“近常数 gate”

这是 Exp-B.1 最重要的正结论。

### 4.1 `w_t` 不再塌成近常数

- `Exp-A w_std_mean ≈ 0.00559`
- `Exp-B w_std_mean ≈ 0.00570`
- `Exp-B.1 w_std_mean ≈ 0.08812`

这几乎是一个数量级的变化。

而且：

- `9/9` 个被试的 `w_std > 0.02`

说明：

> 归一化确实把 gate 从“近常数器”里拉出来了。

### 4.2 尺度压缩确实发生了

按被试平均看：

- 原始几何上下文的平均 std 只有 `~0.027`
- 归一化后平均 std 提升到 `~0.834`

这说明 source-fitted z-score 并不是形式上加了一步，而是**真实改变了输入几何的相对尺度**。

---

## 5. 归一化后，gate 学到了什么？

### 5.1 仍然被 `d_src` 主导

按相关性统计：

- `corr(w, d_src) > 0`: `9/9`
- `corr(w, d_tgt) > 0`: `7/9`
- `corr(w, sigma_recent) > 0`: `7/9`

而且平均相关系数：

- `corr(w, d_src) ≈ 0.480`
- `corr(w, d_tgt) ≈ 0.221`
- `corr(w, sigma_recent) ≈ 0.344`

这说明即使归一化之后，`d_src` 仍然是最稳定的主导信号。

### 5.2 `d_tgt` 并没有成为更强主导项

如果 Exp-B 的主要问题只是尺度，那我们本来会期待：

- `corr(w, d_tgt)` 明显增强
- `w_t` 的结构增强同时带来性能提升

但现在看到的是：

- `w_t` 结构增强了
- `d_tgt` 相关没有明显成为主导
- 平均性能反而下降

这说明：

> 归一化修复了“塌缩”，但没有自动把 gate 变成“更正确的控制器”。

---

## 6. 是否只是尺度问题？

### 结论：**不只是尺度问题。**

这是 Exp-B.1 最重要的最终判断。

因为如果问题**只是**尺度，那么归一化后应该看到：

1. `w_t` 更有结构
2. 平均性能不差于 Exp-B
3. dynamic 至少更稳定地优于 `fixed w=0.5`

但我们实际看到的是：

- 条件 1：成立
- 条件 2：不成立
- 条件 3：不成立

因此最合理的解释是：

> Exp-B 的问题确实包含尺度问题，但**不是只有尺度问题**。  
> 归一化让 gate 变得更敏感了，却没有让它变得更正确。

换句话说，归一化把 gate 从“不会动”变成了“会动”，但没有证明这些变化是在做正确的决策。

---

## 7. 这轮能下什么结论

### 可以下的结论

1. **Exp-B 的“近常数 gate”问题，确实部分来自输入尺度。**
2. **source-fitted z-score 足以让 `w_t` 结构显著增强。**
3. **但 trial-level dynamic gate 的瓶颈不只是尺度问题。**

### 不能下的结论

1. 不能说 Exp-B.1 已经修好了 Exp-B
2. 不能说只要做输入归一化，trial-level gate 就会自然优于 fixed baseline
3. 不能说现在已经可以顺理成章进入 `Exp-C`

---

## 8. 是否继续做 Exp-B.2 或 Exp-C？

### 我的判断：**当前不建议直接进入 Exp-C**

因为现在还没有证据说明：

- richer geometry 已经在 trial-level 上稳定成立
- `w_t` 的变化是“更对”，而不是“更大”

### 对 Exp-B.2 的判断

如果要继续小范围尝试，更合理的是：

- 不是直接加 `rho_window`
- 而是先做一个非常受限的 `Exp-B.2`

例如只做以下一种：

- 固定归一化版本
- 只微调 `bias` 或 `temperature`
- 看能否把结构增益转成性能增益

但若目标是“最快得到能否继续主线的答案”，我更倾向于：

> 先停下来，正式记下这次结果，再重新讨论是继续 trial-level gate 形式，还是把 `rho_window` 作为更强先验接回去。

---

## 9. 正式判断

Exp-B.1 的价值非常明确：

> 它帮助我们排除了一个重要但不完整的解释——  
> **“Exp-B 失败只是因为三维几何特征没做归一化。”**

现在我们知道：

- 归一化能让 gate 动起来
- 但 gate 动起来以后，仍然没有带来更好的主结果

因此下一步不能再简单地说：

> “那就继续加更多信号吧。”

更合理的说法是：

> trial-level dynamic gate 当前的问题已经从“塌缩”升级成了“决策是否正确”。  
> 这意味着后续若继续推进，就必须更明确地回答：  
> **gate 应该被什么信号驱动，才能既有结构，又有正确性。**
