1. 先把 protocol 固死

---

这一部分不要急着写新方法，先把实验协议锁死。

你需要固定这几件事：

1. 数据集与被试划分  
先用主数据集把整套链路跑通；你原文把 BCI Competition IV 视为主战场，并建议后续再扩到更多数据集。  
  

2. 统一预处理  
所有方法必须共享同一套：
    - band-pass / notch
    - epoch 截窗
    - artifact 处理策略
    - 协方差估计方式  
否则 baseline 和新方法无法公平比较。
3. 统一评测协议  
主协议直接用你原文里的 **LOSO**。  

4. 统一随机性  
固定：
    - 3 到 5 个随机种子
    - 所有 split manifest
    - 所有超参数搜索空间  
这样后面做消融时不会“今天赢、明天输”。

3. 方法实现顺序：严格分 4 个版本

---

这一点我建议完全照你文档里“先简单后复杂”的思路执行。你原文已经给出了版本 1/2/3 的降级路线：先只做条件对齐，再做简单反馈，最后才上完整版本。



### V1. 最小可行版
目标：先证明“动态  $ w_t $  优于固定  $ w $ ”。

配置：

+ 标量  $ w_t $ 
+ 对齐形式：选项 A
+ 上下文只用纯几何特征  
$ [d_{\text{geo}}, d_{\text{src}}, d_{\text{tgt}}, \sigma_{\text{recent}}] $ 
+ gate 用最简单 MLP 或线性-sigmoid
+ 只做离线训练，不做在线伪标签微调  
这正对应你原文推荐的最稳版本。  
  
  


### V2. Koopman/KCAR 版
目标：验证你的方案 B 是否比普通动态 gate 更好。

配置：

+ 在 V1 基础上加入  $ \rho_t $  / KCAR
+ gate 输入从纯几何扩成  
$ [d_{\text{src}}, d_{\text{tgt}}, \sigma_t, H_{t-1}, p^{\max}_{t-1}, \rho_t] $ 
+ 仍然不做伪标签自训练  
这一步是你当前最值得押的主方法版本。

### V3. 在线安全更新版
目标：验证“在线适应”而不是只做静态 LOSO。

配置：

+ 在 V2 基础上加入滑窗 target 状态更新
+ 仅在低风险 + 高置信度条件下允许伪标签更新
+ 输出按 trial 的在线曲线  
这和你原文的在线阶段设定是吻合的：先估计目标中心，再逐 trial 推理，必要时做保守更新。  


### V4. 完整增强版
只在 V2/V3 已经有效时再做：

+ 加行为信号  $ H_t, conf_t, acc\_trend $ 
+ 加反馈规则 1，再视情况加规则 2/3
+ 尝试仿射变换 B
+ 最后再碰 MLP 对齐 C  
你原文对这个顺序也写得很明确：反馈先从规则 1 开始，仿射/MLP 放后面。
