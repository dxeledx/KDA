## 一句话结论

`KSDA v3` 当前已经得到一个非常清晰的中间结论：

> **动态需求存在，但当前“局部专家集”还没有提供足够互补的动作空间。**

也就是说：
- `D0` 明确支持“这条线值得动态化”
- `D1` 明确说明“当前这组专家还不足以支撑下一步的信号学习”
- 因此当前不进入 `D2 / D3`

---

## 1. D0 回答了什么

`D0` 只回答一个问题：

> **测试序列上，是否真的存在“局部最优动作随时间变化”的证据？**

### 结果

最佳窗口大小为 `16`，对应结果：

| 指标 | 数值 |
|------|------|
| `oracle_accuracy_mean` | `0.4502` |
| `best_fixed_w` | `1.0` |
| `best_fixed_accuracy_mean` | `0.4093` |
| `mean_delta_vs_best_fixed` | `+0.0409` |
| `wins_vs_best_fixed` | `9/9` |
| `most_common_ratio` | `0.3395` |
| `subjects_with_multiple_actions` | `9/9` |

而且这不是只在单一窗口大小上偶然成立：
- `window=16`：`+4.09%`
- `window=32`：`+2.78%`
- `window=48`：`+2.24%`

三个窗口都通过了我们预设的 `D0` 门槛。

### 结论

`D0` 给出的结论非常强：

> **局部动态需求是存在的。**

这意味着你现在已经可以明确排除一种解释：

> 问题不是“根本不需要动态化”，而是“我们还没有找到合适的动态动作空间”。  

这一步很重要，因为它把后续所有工作都重新定性了：
- 不是去怀疑“要不要做动态”
- 而是去问“该用什么动作空间做动态”

---

## 2. D1 回答了什么

`D1` 回答的是：

> **在当前 Koopman 表征上，预先构造的局部线性专家集是否足够互补，能否支撑动态选择？**

当前专家集是：
- `E0`: identity
- `E1`: conservative mean shift
- `E2`: diagonal scaling
- `E3`: low-rank shrinkage affine
- `E4`: supervised subspace aligner

### 结果

`D1` 的 summary 是：

| 指标 | 数值 |
|------|------|
| `best_single_expert` | `E0_identity` |
| `best_single_accuracy_mean` | `0.4109` |
| `oracle_accuracy_mean` | `0.4232` |
| `mean_delta_vs_best_single` | `+0.0123` |
| `wins_vs_best_single` | `9/9` |
| `most_common_ratio` | `0.8272` |
| `subjects_with_multiple_experts` | `9/9` |
| `max_single_subject_contribution` | `0.2188` |
| `passed` | `False` |

### 为什么没通过

虽然 `oracle-expert` 的平均准确率比最佳单专家高了 `+1.23%`，而且是 `9/9` 被试都占优，但它仍然**没有通过**，原因非常明确：

> **最优专家分配塌缩到了 `E0_identity`。**

更具体地说：
- `E0_identity` 占了 `82.7%` 的窗口
- `E1_mean_shift` 与 `E2_diag_scaling` 的全局表现甚至和 `E0` 完全一样
- `E3` 与 `E4` 只在少量窗口被选中

这意味着当前 D1 观测到的不是：

> “状态在多个局部线性策略之间稳定切换”

而更像是：

> “identity 已经覆盖了大多数窗口，其他专家只在少量局部补边”

所以这一步不能被解释成“动态专家集已经成立”，只能解释成：

> **当前专家集有一点互补性，但互补性还不够强，动作空间仍然偏塌缩。**

---

## 3. D0 与 D1 合在一起说明了什么

这是这轮最关键的地方。

### D0 的结论

> 数据里确实存在值得动态化的局部决策结构。

### D1 的结论

> 当前构造出来的专家集还没有把这种局部决策结构表达出来。

把两者放在一起，得到的不是矛盾，而是一个非常有价值的诊断：

> **动态需求是真实存在的，但当前动作空间定义得还不够好。**

这和你最初的研究想法是高度一致的：
- 你要做的不是证明“动态本身需不需要”
- 而是找到一个能承载动态选择的 Koopman 空间动作集

所以当前最合理的研究判断是：

1. **这条动态主线值得继续**
2. **但下一步优先级不该是直接做 `D2/D3`**
3. **而是先把 `D1` 的专家集重构得更有互补性**

---

## 4. 当前可以支持什么，不能支持什么

### 可以支持的结论

1. **局部动态需求成立**
2. **简单的固定单动作不足以解释最优表现**
3. **当前专家集不是完全无用，但明显过于偏向 `identity`**
4. **当前最该优化的是动作空间，不是马上学更复杂的信号**

### 不能支持的结论

1. 不能说 **动态专家集已经成立**
2. 不能说 **现在就该进入 `D2` 的线性性能指标学习**
3. 不能说 **当前 `KCAR` 或其他 proxy 已经准备好接入**

---

## 5. 正式决策

### 本轮决策

**不进入 `D2 / D3`。**

### 原因

不是因为动态主线失败，而是因为：

> **`D1` 没有通过“专家集不塌缩”的门槛。**

继续往 `D2 / D3` 推，只会把问题重新耦合成：
- 是信号不行？
- 还是专家集不行？

这正是 v3 主线想避免的事情。

---

## 6. 当前最合理的研究口径

建议把当前阶段的口径固定成下面三句话：

1. `D0` 已经证明：**局部动态需求真实存在。**
2. `D1` 已经证明：**当前专家集还不能充分表达这种局部动态结构。**
3. 因此下一步不是 `D2/D3`，而是**重构 `D1` 的动作空间**。  

---

## 7. 对下一步的建议

当前最值得做的不是更复杂的 controller，而是更好的专家集。

优先方向建议：

### 方向 A：让专家真正“拉开”

当前问题是：
- `E1_mean_shift` 和 `E2_diag_scaling` 几乎等价于 `identity`
- `E3/E4` 只在少量窗口起作用

所以下一步应优先提升专家间差异性，而不是继续加 gate。

### 方向 B：保留 D0 的窗口级 oracle 作为校准器

新的专家集出来后，先重复 `D1` 这一步：
- 看 oracle 是否仍显著优于单专家
- 更重要的是看是否不再塌缩到单一专家

只有这一层站住，`D2` 的线性性能指标学习才有意义。

---

## 最终判断

如果只保留一句最重要的话：

> **`D0` 证明了“动态是必要的”，`D1` 证明了“当前专家集还不够好”；因此 KSDA v3 的下一步应是重做动作空间，而不是急着进入信号学习或算子修正。**
