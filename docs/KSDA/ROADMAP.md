# KSDA实验路线图总结

**创建日期**: 2026-03-08
**目标**: 通过Koopman空间动态对齐突破当前困境，冲击顶会论文

---

## 核心思想

将跨被试对齐从**协方差流形空间**转移到**Koopman线性空间**，实现三个递进创新:

1. **Koopman空间对齐**: 利用线性结构，避免SPD流形复杂性
2. **线性结构的动态适应**: 在线更新更高效，理论分析更清晰
3. **性能驱动的Koopman修正**: 用分类性能反馈修正动力学估计

---

## 实验路线 (4个阶段)

### Phase 1: Exp-D.1 - 静态Koopman空间对齐 (1-2周)

**目标**: 验证基础可行性

**方法**:
```
C_t → Koopman特征 ψ_t → 静态对齐 A @ ψ_t → 分类
```

**验收标准**:
- ✅ 性能 ≥ RA baseline (43.4%)
- ✅ 计算时间 < RA

**如果失败**: 整个KSDA方向终止

---

### Phase 2: Exp-D.2 - 在线更新对齐矩阵 (2-3周)

**前置**: Exp-D.1成功

**方法**:
```
在线更新 A_t (OGD或EMA)
高置信度时更新，低置信度时保持
```

**验收标准**:
- ✅ 性能 > KSDA-static + 1%
- ✅ A_t确实在适应

**如果失败**: 回到静态版本，分析原因

---

### Phase 3: Exp-D.3 - 性能驱动的Koopman修正 (2-3周)

**前置**: Exp-D.2成功

**方法**:
```
根据在线性能加权更新 K_t
性能好 → 高权重，性能差 → 低权重
```

**验收标准**:
- ✅ 性能 > KSDA-online + 1%
- ✅ 动力学质量改善

**如果失败**: 保留D.1-D.2，放弃性能反馈

---

### Phase 4: Exp-D.4 - KCAR驱动的风险感知对齐 (1-2周)

**前置**: Exp-D.3成功

**方法**:
```
用KCAR控制对齐强度 w_t
ρ_t高(高风险) → w_t低(弱对齐)
ρ_t低(低风险) → w_t高(强对齐)
```

**验收标准**:
- ✅ 性能 > KSDA-perf + 1%
- ✅ 性能 ≥ RA baseline
- ✅ 优于stage3窗口级KCAR

**如果成功**: 完整KSDA框架，准备投稿

## 与现有工作的对比

### Stage2 (DCA-BGF)
- 协方差空间对齐
- 几何特征驱动gate
- **问题**: gate决策逻辑不清晰，性能未超baseline

### Stage3 (窗口级KCAR)
- 协方差空间 + KCAR诊断
- 窗口级离散策略
- **问题**: 诊断与对齐分离，未充分利用Koopman结构

### KSDA (新方案)
- Koopman空间对齐
- 动力学驱动 + 性能反馈
- **优势**: 统一框架，端到端优化，理论保证

---

## 关键优势

### 理论优势
1. **计算复杂度**: O(m²) vs O(n³)，快10-100倍
2. **在线学习**: 线性空间可证明regret bound
3. **动力学一致性**: 对齐保持谱结构

### 实践优势
1. **更高效**: 避免矩阵分解，纯线性运算
2. **更统一**: KCAR和对齐在同一空间
3. **更灵活**: 支持在线更新和元学习

---

## 风险与应对

### 风险1: Exp-D.1失败 (Koopman空间对齐不如协方差空间)

**概率**: 中等 (30%)

**应对**:
- 分析是lifting问题还是本质问题
- 如果是lifting，尝试学习MLP
- 如果是本质问题，回到stage3

### 风险2: 在线更新不稳定

**概率**: 低 (20%)

**应对**:
- 降低学习率
- 增加正则化
- 使用更保守的更新策略

### 风险3: 性能反馈噪声太大

**概率**: 中等 (30%)

**应对**:
- 增大性能窗口
- 使用更平滑的性能估计
- 只在高置信度时更新

### 风险4: KCAR在Koopman空间无效

**概率**: 低 (20%)

**应对**:
- 调整KCAR窗口大小
- 尝试不同门控函数
- 或接受D.1-D.3的部分成功
