## 1. 实验目的

这是v2.0路线中**最关键的实验**,要回答:

> KCAR作为风险信号,是否明显优于几何特征?

**与Exp-B.1的对比**:
- Exp-B.1: 归一化后的几何特征 (d_src, d_tgt, sigma)
- Exp-D.1+: KCAR (动力学一致性)

**核心假设**:
- 几何特征只告诉你"当前在哪里" (间接)
- KCAR告诉你"源域动力学能否解释当前演化" (直接)
- KCAR应该是更好的信号

**如果失败**: 说明问题不在信号,可能在方法本身

## 2. 方法设计

### 2.1 Pipeline

```
输入: 协方差序列 {C_t}

初始化:
  K_s = fit_global_koopman(source_data)  # 源域算子 (固定)
  K_t = K_s.copy()                       # 目标域算子 (固定)
  A = learn_alignment_matrix(source_data) # 对齐矩阵 (固定)

在线循环 (t = 1, 2, ..., T):
  Step 1: Koopman Embedding
    z_t = tangent_project(C_t, M_s)
    ψ_t = lift_quadratic(z_t)

  Step 2: Compute KCAR (关键)
    # 用最近窗口计算KCAR
    window = history[-32:]
    e_src = compute_residuals(window, K_s)
    e_tgt = compute_residuals(window, K_t)
    ρ_t = (e_src - e_tgt) / (e_src + e_tgt + ε)

  Step 3: KCAR-driven Gate (简单版)
    w_t = sigmoid(a - b * ρ_t + c)  # b > 0

  Step 4: Alignment in Koopman Space
    ψ'_t = (1 - w_t) * ψ_t + w_t * (A @ ψ_t)

  Step 5: Classification
    y_pred = LDA(ψ'_t)
```

**关键**: A和K_t都是固定的,只测试KCAR作为信号

---

### 2.2 KCAR门控函数

**方案A: 简单线性门控** (推荐首选)

```python
w_t = sigmoid(a - b * ρ_t + c)

# 初始参数:
a = 1.0   # 基础对齐倾向
b = 2.0   # KCAR敏感度 (b > 0)
c = 0.0   # 偏置

# 性质:
# ρ_t > 0 (高风险) → w_t 低 → 弱对齐
# ρ_t < 0 (低风险) → w_t 高 → 强对齐
```

**方案B: 分段线性门控** (如果A不够)

```python
if ρ_t > 0.3:
    w_t = 0.5  # 高风险,部分对齐
elif ρ_t < -0.3:
    w_t = 1.0  # 低风险,完全对齐
else:
    w_t = 0.7  # 中等风险
```

---

### 2.3 关键设计决策

**决策1: KCAR窗口大小**
- 当前: 32 trials (与stage3保持一致)
- 可以尝试: 16, 64

**决策2: K_t是否更新**
- 第一版: K_t = K_s (固定)
- 如果不够: K_t用滑窗更新 (但不用性能反馈)

**决策3: 门控参数**
- 第一版: 手工设定 (a=1.0, b=2.0, c=0.0)
- 如果不够: 在源域上学习

**决策4: 是否加平滑项**
- 第一版: 不加 (保持简单)
- 如果w_t震荡: 加入 λ * (w_t - w_{t-1})²

## 5. 验收标准

### 5.1 情况A: KCAR明显优于几何特征 ✅

**证据**:
1. **性能提升**:
   - KSDA-kcar-gate > KSDA-geometric-gate + 2%
   - 在7/9个被试上优于geometric-gate

2. **KCAR有效性**:
   - corr(ρ_t, performance) > corr(d_tgt, performance)
   - ρ_t能预测未来性能变化

3. **决策质量**:
   - ρ_t与w_t负相关 (验证逻辑)
   - w_t的变化与性能改善相关

**行动**:
→ 继续Exp-D.2 (在线更新A_t)
→ 继续Exp-D.3 (性能反馈K_t)
→ 完整KSDA框架

---

### 5.2 情况B: KCAR略优于几何特征 🟡

**证据**:
1. **性能提升**:
   - KSDA-kcar-gate > KSDA-geometric-gate + 0.5-1%
   - 但仍未超越RA baseline

2. **KCAR有效性**:
   - ρ_t与性能有相关性,但不强
   - 优势不够明显

**行动**:
→ 不继续D.2/D.3 (在线更新可能无法弥补差距)
→ 转向**窗口级KCAR策略**

**窗口级策略**:
```python
# 不是trial-level连续控制
# 而是窗口级离散决策

# 每32个trial计算一次ρ_window
ρ_window = compute_kcar(window=32)

# 离散动作
if ρ_window > 0.3:
    action = "use w=0.5 for next 32 trials"
elif ρ_window < -0.3:
    action = "use w=1.0 for next 32 trials"
else:
    action = "use w=0.7 for next 32 trials"
```

## 9. 失败应对方案

### 如果KCAR不优于几何特征

**诊断步骤**:

1. **检查KCAR本身**
   - ρ_t的分布是否合理?
   - ρ_t是否有变化? (不是常数)
   - ρ_t与几何特征的相关性?

2. **检查Koopman算子质量**
   - K_s和K_t的预测误差
   - 是否K_s和K_t都不准确?

3. **检查门控函数**
   - w_t的分布是否合理?
   - 门控参数(a,b,c)是否合适?

**修复尝试**:

**尝试1: 调整KCAR窗口**
```python
# 当前: window=32
# 尝试: window=16 (更敏感), window=64 (更平滑)
```

**尝试2: 改进Koopman算子**
```python
# 当前: 全局K_s
# 尝试: 类条件K_s^(y)

for each class y:
    K_s^(y) = fit_koopman(source_data[y])

K_s_weighted = Σ p(y|ψ_t) * K_s^(y)
```

**尝试3: 改变门控形式**
```python
# 当前: 连续sigmoid
# 尝试: 分段线性或离散决策
```

**尝试4: 窗口级KCAR**
```python
# 不是trial-level
# 改为窗口级决策
```

**止损规则**:
- 如果尝试了1-4都失败
- 且窗口级KCAR也无效
- 则承认KCAR在当前setting下不可行
- 回到stage3深化分析

## 11. 与Exp-B.1的对比

### 相同点
- 都是验证信号的有效性
- 都用固定的对齐方法
- 都是关键决策点

### 不同点

| 维度 | Exp-B.1 | Exp-D.1+ |
|------|---------|----------|
| 空间 | 协方差空间 | Koopman空间 |
| 信号 | 几何特征 | KCAR |
| 对齐 | RA (协方差) | A (Koopman) |
| 目标 | 验证归一化 | 验证KCAR |

### Exp-B.1的教训

**不要重复的错误**:
- 不要只看w_t的结构,要看性能
- 不要假设"有结构=正确"
- 要直接对比信号的预测能力

**要保留的优点**:
- 严格的单变量控制
- 详细的分析和可视化
- 清晰的验收标准
